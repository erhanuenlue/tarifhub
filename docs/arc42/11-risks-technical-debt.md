# 11 · risks technical debt

The risk register tracks each known risk with its mitigation and residual exposure. Every row is grounded in a dated incident from the contemporaneous record (`vault/daily/`, `LEARNINGS.md`, the decision matrix) and names its evidence, so this is an honest accounting and not a checklist.

| Risk | Mitigation | Residual |
|---|---|---|
| **AI fill variance**: live `ai_map` fills are not byte-stable across re-ingests (measured 2026-06-12: 55 re-versions of ATC-less SL records over two re-runs), breaking re-ingest idempotency | Fill-reuse: unchanged content carries the prior version's fills forward with no model call ([ADR-005 addendum](../adr/005-single-ai-seam.md)); live-measured, a full-export reuse leg froze 0 of 10 299 with an invalid key ([proof](../evidence/2026-06-12-sl-live-ingest.md#addendum-2026-06-12-live-fill-reuse-proof)) | `--refill` and model upgrades deliberately re-version (append-only); accepted, since each variant is audit-logged and routes to review |
| **Silent scale rounding**: Postgres `NUMERIC(12,4)/(12,2)` rounds an over-precise billing value on insert, so stored bytes ≠ hashed bytes | Pre-freeze quantization to the canonical scales; lossy values fail closed to `None` + review ([ADR-016](../adr/016-decimal-scale-contract.md)) | Closed: stored bytes provably equal hashed bytes on every engine |
| **Engine parity (SQLite mirror vs Postgres)**: a whole class of bugs surfaces only on the real engine, an `int` written into a BOOLEAN column (rejected by psycopg, accepted by SQLite) and `json.loads()` on a Postgres JSONB value (already a dict, would have 500'd every read); both passed dozens of green SQLite tests (2026-06-11, PR #2, PR #6) | The cross-engine read-parity suite asserts every read endpoint's full JSON body against an engine-independent snapshot on both SQLite and Postgres, so the two must match each other; the Postgres leg runs against a real `pgvector/pgvector:pg16` container in the CI `python-parity` job ([§13](13-test-strategy.md)) | The offline default suite cannot, by construction, prove Postgres-only behaviour; this is bounded by the Postgres-gated parity job that runs the identical tests against the real engine on every push and PR |
| **Console test wiring** (register corrected 2026-06-14): an earlier revision of this row described the Vitest component specs and offline Playwright smoke as unwired (a no-op `npm run test --if-present`); that was inaccurate | In fact the `test` script (`vitest run && playwright test`) has been wired since the console landed (PR #16, commit `265b2e3`, 2026-06-13), and the CI `console` job runs it on every push to `main` and every pull request: 18 Vitest tests across four files plus a Playwright end-to-end smoke ([§13](13-test-strategy.md#console-component-tests), [ADR-013](../adr/013-demo-scope.md)) | None: the specs that assert the brand visual law execute in CI; the latest `main` run is 18 Vitest passed and the Playwright smoke passed (a `CAPTURE`-gated screenshot spec is skipped) |
| **Search availability**: semantic search returns HTTP 501 when the e5 backend is absent on Postgres (a dimension mismatch against the `vector(1024)` column), by design | Honest unavailability: the path fails closed with an explicit 501 before issuing a doomed query rather than faking a ranking ([ADR-017](../adr/017-deterministic-search-fallback-explain.md)); offline on SQLite, the same query path ranks via a deterministic in-process cosine over the stored stub embeddings, so UC-06 stays fully testable | Real semantic search (cross-lingual e5 ranking on pgvector) needs the e5 backend to be exercised; the offline cosine proves the contract shape, not e5 retrieval quality |
| **FR ranking trade-off**: the e5-correct `query:` prefix lifted cross-lingual recall@5 from 0.833 to 0.917 (both Italian misses recovered, French `hématocrite` 3 → 2) but introduced an MRR@5 dip from 0.681 to 0.597 as several rank-1 hits slipped to rank 2 | The trade-off is documented, not hidden: the dip is mostly cosmetic same-name `.01` billing-variant swaps, and the query prefix ships because it is the e5-correct usage and cross-lingual recall is the defect Block-0 exposed ([decision-matrix](../method/decision-matrix.md), `LEARNINGS.md`, PR #7, full tables in [evidence](../evidence/2026-06-11-fr-ranking-eval.md)) | A documented follow-up stays open: extend passage text with FR/IT designations and re-embed the corpus so `hématocrite → 1375` reaches rank 1 |
| **L2 intelligence scope**: the TarifIQ intelligence service (frozen rule table, combinability evaluation, TARMED↔TARDOC crosswalk) is containerised and offline-tested but is post-CAS scope and outside the graded MVP value path | The boundary is stated explicitly: the graded MVP value path is L0 ingestion → L1 serving/MCP → L3 console; L2 is read-only and post-freeze (it reads frozen facts from L1), packaged like every other sub-system so it can be brought up for completeness without touching the value path ([ADR-013](../adr/013-demo-scope.md), [§5](05-building-block-view.md)) | None for the MVP; the row exists to mark the boundary and avoid overclaiming the intelligence feature as delivered scope |

Overall debt posture: the load-bearing risks on the value path (fill variance, scale rounding, engine parity) are closed or bounded by a named, CI-enforced mitigation, the console test wiring runs in CI (an earlier overstatement in this register, that the specs were unwired, is corrected above), and the remaining open items (e5-only search quality, L2 scope) are recoverable and flagged on purpose rather than discovered later.
