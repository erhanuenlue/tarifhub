## Total estimate: **94 / 100**

The build is in strong shape: every structural element is present and the prose backing it is,
in almost every case, specific and evidenced rather than generic. The estimate sits below 100
on exactly two judgment calls, both honest and both owner-gated by design:

1. **Criterion 15 (−5):** the *Erklärung der Eigenständigkeit* is a **self-declared visible
   placeholder** ("It is not signed here, and the paraphrase below is not yet his words …
   Awaiting the owner's final text and signature"). The 12-point anchor requires it
   *"gemäss Aufgabenstellung vollständig"*. By the conjunctive rule, an unfinished, unsigned
   declaration drops the tier even though every other crit-15 anchor (per-phase structure,
   prompts/diffs/commit refs) is fully met. This is owner-gated by Fazit veto 3, so it is
   *intentionally* not yet final, but it is the gating element today.
2. **Criterion 8 (−1):** framework usage is idiomatic and the four named concepts are real in
   code, but the conjunctive "**durchgehend** idiomatisch" anchor is shaded by one
   self-disclosed inconsistency: the console's Vitest/Playwright specs exist but are
   "not yet wired to a `package.json` test script", so the CI `console` job runs
   `npm run test --if-present` as a no-op. Small, disclosed, but it is a gap against "throughout".

Everything else is judged at full marks against quoted evidence below.

## Gap list — ranked by points at risk

### 1. Criterion 15 — Eigenständigkeitserklärung is a placeholder (−5, highest leverage)
The §15 chapter is otherwise at the 12-point bar: the four phases (Generierung / Review /
Refactoring / Recherche) are each present and **each claim is tied to a prompt line, a file/diff,
and a PR + commit SHA** ("FILL-ONLY: set designation.fr/it/category ONLY where the current value
is None … (PR #2, `057a6c1`)"; the Review section's two-disposition Codex P1/P2 example
`ac8296c`; the drift-guard → `test_model_contract.py` accept `257c0e0`). The single conjunctive
miss is the declaration itself, which the document openly marks:
> "Editorial note: this section is a visible placeholder … It is not signed here, and the
> paraphrase below is not yet his words … Awaiting the owner's final text and signature."
The 12-point anchor demands "Erklärung der Eigenständigkeit **gemäss Aufgabenstellung vollständig**".
A blank Name/Ort/Datum/Unterschrift table and a paraphrased-not-final body fails "vollständig",
so the tier drops to *überwiegend* (7). **This is by design owner-gated (Fazit veto 3) and only
becomes final at submission** — but as of today it is the single biggest correctable delta.
**Securing the 5 points:** owner authors and signs the final FFHS declaration verbatim before the
PDF hand-in; remove the placeholder banner.

### 2. Criterion 8 — "durchgehend idiomatisch" shaded by un-wired console tests (−1)
Server-side framework usage is genuinely idiomatic and quoted from code:
`RepoDep = Annotated[ServingRepository, Depends(get_repository)]`, per-request connection
lifecycle with `finally: conn.close()`, declarative `response_model`, boundary-only
`HTTPException(404)` with structured `detail`, env-only `get_settings()` config. §8 also quotes
criterion 8 verbatim and maps each named concept (DI, REST/OpenAPI, Konfiguration,
Fehlerbehandlung) to a concrete file. The one crack in "throughout": §5 self-discloses that the
console's Vitest + Playwright specs "are not yet wired to a `package.json` test script, so the CI
`console` job … runs `npm run test --if-present` as a no-op." Idiomatic-but-untested on one
sub-system keeps it at *überwiegend* (9) rather than *vollständig* (10).
**Securing the point:** wire `apps/tarifguard/tests` + `e2e` into a real `npm test` script and
let the CI `console` job execute them (the specs already exist).
